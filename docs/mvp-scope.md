# MVP Scope (Locked)

This document defines the functional scope for MVP delivery.  
Any feature outside this list is deferred to Phase 2.

## MVP Priority (Build now)

- Public online reservations flow
- Real availability checks
- Safe table assignment
- Basic admin panel operations
- Manual blocks management
- Opening hours management
- Confirmation emails

## Phase 2 (Do not build now)

- WhatsApp messaging
- Deposits or payments
- Waitlist
- Promo codes
- Loyalty program
- Advanced analytics

## Delivery rule

When implementation trade-offs appear, choose the smallest robust solution that protects:

1. Availability accuracy
2. Double-booking prevention
3. Mobile usability for booking flow
4. Operational clarity for staff
